# 项目说明

## 1. 项目定位

Wesi3D 是一个面向 `SEG-Y` 三维地震体数据的桌面可视化与处理项目。
当前版本使用 `Python + PySide6 + VTK + segyio` 实现，重点能力包括：

- 读取大型 `SEG-Y` 体数据
- 对体数据进行降采样与范围提取
- 在 3D 场景中显示正交切片
- 提取连通体形成地层/包络层
- 提取、编辑、回写控制点

这次重构的目标不是一次性重写业务逻辑，而是先把项目整理成适合继续开发的结构：源码、数据、脚本、文档分离，入口明确，数据目录分层清晰。

## 3. 每个目录的作用

### 3.1 `src/`

正式业务源码目录。后续所有真正参与项目运行的代码都应优先放在这里，而不是继续平铺在项目根目录。

### 3.2 `src/wesi3d/`

主包目录，表示项目的正式 Python 包。

### 3.3 `src/wesi3d/app/`

界面与应用组织层。
主要负责：

- 程序入口参数解析
- Qt 窗口与交互面板
- VTK 视图初始化
- 将数据层和处理层结果接入界面

### 3.4 `src/wesi3d/data/`

数据访问与渲染前的数据准备层。
主要负责：

- 读取 `SEG-Y`
- 管理体数据结构
- 把体数据转换为 VTK 可显示的数据对象

### 3.5 `src/wesi3d/processing/`

体数据处理层。
主要负责：

- 体数据范围裁剪
- 降采样
- 连通组件提取
- 控制点提取、编辑、重建

### 3.6 `src/wesi3d/utils/`

通用工具层。
只放跨模块复用的轻量工具，例如：

- 常量定义
- 数值显示格式化

### 3.7 `data/`

项目数据目录，按用途分层，避免原始数据、样例数据、派生数据混在一起。

### 3.8 `data/raw/`

原始输入数据。
用于存放未经处理的 `SEG-Y` 文件，例如：

- `vel.sgy`
- `stk.sgy`

约定：

- 原始数据只进不改
- 不在这里放导出结果
- 默认不纳入 Git 版本控制

### 3.9 `data/derived/`

派生数据目录。
用于存放处理结果，例如：

- 裁剪后的小体积数据
- 提取出的属性结果
- 后续可缓存的中间结果

约定：

- 这里的数据可重复生成
- 默认不纳入 Git

### 3.10 `data/samples/`

样例数据目录。
用于放小尺寸、可分发、适合测试或演示的数据。

适合放：

- 小型测试 SEG-Y
- 单元测试夹具数据
- 演示用数据片段

### 3.11 `scripts/`

辅助脚本目录。
这里放实验脚本、环境验证脚本、一次性工具脚本，不放正式业务入口。

### 3.12 `scripts/smoke/`

冒烟测试脚本目录。
用于验证本机 GUI 或环境是否正常。

### 3.13 `scripts/prototypes/`

原型脚本目录。
用于保留早期验证思路、算法演示、渲染实验，不直接参与正式业务流程。

### 3.14 `docs/`

项目文档目录。
后续建议放：

- 架构说明
- 数据格式说明
- 开发约定
- 重构记录

## 4. 每个程序的作用

### 4.1 `main.py`

根目录启动脚本。
作用：

- 为本地开发提供稳定入口
- 自动把 `src/` 加入 Python 模块搜索路径
- 转发到 `wesi3d.main`

建议启动方式：

```powershell
.\.venv\Scripts\python.exe main.py
```

### 4.2 `src/wesi3d/main.py`

包内入口。
作用：

- 作为正式应用入口转发层
- 对外暴露 `main()`
- 将来如果要做打包或命令行入口，这里是首选挂载点

### 4.3 `src/wesi3d/config.py`

默认配置定义。
作用：

- 定义项目根目录
- 定义 `data/raw`、`data/derived`、`data/samples` 路径
- 保存查看器默认参数，例如默认 SEG-Y 路径、采样间隔、步长、透明度等

### 4.4 `src/wesi3d/app/viewer.py`

主查看器程序，是当前项目最核心的业务文件。
作用：

- 解析命令行参数
- 加载 SEG-Y 数据
- 创建 VTK 场景与 Qt 界面
- 管理切片、属性、地层、控制点的交互流程

说明：

- 当前它仍然是一个较大的文件
- 这是本轮重构刻意保留的结果，优先保证可运行和职责归位
- 后续可以再拆成 `dialogs.py`、`widgets.py`、`scene.py` 等子模块

### 4.5 `src/wesi3d/data/volume_data.py`

底层体数据模块。
作用：

- 读取 `SEG-Y` 数据
- 组织 `xline / inline / sample` 三轴数据
- 管理几何信息、缓存与大体积访问逻辑

### 4.6 `src/wesi3d/data/attribute_data.py`

渲染数据适配模块。
作用：

- 把 `VolumeData` 转成 `vtkImageData`
- 构建颜色查找表
- 为显示层提供可直接渲染的属性对象

### 4.7 `src/wesi3d/processing/volume_processing.py`

体数据处理模块。
作用：

- 范围提取
- 体数据复制与降采样
- 连通组件提取
- 包络体构建

### 4.8 `src/wesi3d/processing/control_points.py`

控制点处理模块。
作用：

- 从 horizon / mask 提取控制点
- 管理主控制点与普通控制点关系
- 根据控制点位移重建结果

### 4.9 `src/wesi3d/utils/constants.py`

常量模块。
作用：

- 放项目内共享的固定值
- 当前主要包含 SEG-Y 头字段常量和应用名称

### 4.10 `src/wesi3d/utils/formatting.py`

格式化模块。
作用：

- 放共享的显示格式化函数
- 当前主要负责数值字符串格式化

### 4.11 `scripts/smoke/tkinter_smoke_test.py`

GUI 冒烟测试程序。
作用：

- 验证本机 `tkinter` 是否可正常弹窗和响应控件
- 与正式查看器逻辑无耦合

### 4.12 `scripts/prototypes/vtk_pyside_prototype.py`

渲染原型程序。
作用：

- 保存早期 `PySide6 + VTK` 原型验证代码
- 使用模拟体数据演示 3D 等值面渲染
- 用于实验，不作为正式入口

### 4.13 `requirements.txt`

依赖声明文件。
当前依赖包括：

- `numpy`
- `PySide6`
- `segyio`
- `vtk`

## 5. 数据分割约定

为了避免后续开发越来越乱，数据目录建议严格按下面规则使用：

### 5.1 原始数据

放在 `data/raw/`。
特点：

- 来源清晰
- 保持原始状态
- 体积通常很大
- 不要手工覆盖修改

### 5.2 派生数据

放在 `data/derived/`。
特点：

- 由程序生成
- 可以删除后重建
- 适合缓存中间结果和导出结果

### 5.3 样例数据

放在 `data/samples/`。
特点：

- 体积小
- 便于测试、调试、演示
- 可以考虑纳入版本控制

### 5.4 默认启动数据

当前查看器默认读取：

```text
D:\Wesi3D\data\raw\vel.sgy
```

如果要切换数据，可以：

```powershell
.\.venv\Scripts\python.exe main.py data\raw\stk.sgy
```

## 6. 建议的开发约定

### 6.1 新代码放置规则

- 正式功能代码放 `src/wesi3d/`
- 原型验证代码放 `scripts/prototypes/`
- 环境验证脚本放 `scripts/smoke/`
- 数据文件不要再直接放项目根目录

### 6.2 后续拆分建议

下一轮最值得拆的文件是：

- `src/wesi3d/app/viewer.py`

建议拆分方向：

- `dialogs.py`：弹窗
- `widgets.py`：控件封装
- `scene.py`：VTK 场景构建
- `viewer_window.py`：主窗口

### 6.3 测试优先级

建议优先为以下模块补测试：

- `src/wesi3d/data/volume_data.py`
- `src/wesi3d/processing/volume_processing.py`
- `src/wesi3d/processing/control_points.py`

因为这些模块算法性强，且相对容易脱离 GUI 做验证。
